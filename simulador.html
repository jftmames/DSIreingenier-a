Laboratorio de Rediseño DSI

Reto Interactivo: Optimización de Costes de Inferencia y Gobernanza

Paso 1: Diagnóstico del Problema

Contexto: Habéis implementado un chatbot con IA generativa (LLM) para atender los tickets de soporte TI. El uso ha crecido rápido, pero el sistema está colapsando: el coste mensual de inferencia es inasumible y los usuarios se quejan de que el bot tarda mucho en responder [cite: 578-580].

Prompt para el equipo: Como diseñadores de sistemas, ¿cuál debe ser vuestra primera decisión para abordar este problema operativo?

Paso 2: La Calidad de Entrada

Análisis: Habéis descubierto que el coste de inferencia es alto porque las entradas de los usuarios son muy pobres ("no me va el sistema"). El LLM gasta miles de tokens y tiempo interactuando para averiguar qué sistema falla y cuál es el error [cite: 553-556].

Prompt para el equipo: ¿Cómo rediseñáis la entrada del proceso (TO-BE)?

Paso 3: Segmentación de Rutas (El núcleo tecnológico)

Análisis: El 60% de los tickets son consultas simples tipo "¿Cómo configuro la VPN?", que ya están en los manuales de la empresa. El sistema actual usa el LLM para redactar una respuesta nueva cada vez.

Prompt para el equipo: ¿Qué arquitectura de decisión aplicáis para optimizar el coste y el tiempo?

Paso 4: Gobernanza y Supervisión

Análisis: El sistema ya es eficiente. Sin embargo, en ocasiones entra un ticket clasificado como "Brecha de seguridad en base de datos de clientes".

Prompt para el equipo: ¿Qué papel juega la IA y el ser humano aquí?
